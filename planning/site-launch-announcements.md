# FRESH Lab Website Launch Announcements

Date drafted: 2026-06-28

## Basecamp Message Board Draft

Subject: New FRESH lab website is live - please help review

Hi everyone,

Big FRESH lab milestone: our new lab website is now live.

New site:

https://ubc-fresh.github.io/fresh-lab-web-site/

This is a much more maintainable, version-controlled website than the old UBC CMS/WordPress site. The site source now lives in GitHub, so updates can be reviewed, tracked, and improved over time instead of being trapped inside a browser-only WordPress editor.

Please click through the whole website when you have a few minutes and look for anything that seems wrong, missing, stale, awkward, broken, or confusing:

- broken links;
- typos;
- weird layout on desktop or mobile;
- missing or incorrect names;
- incorrect project descriptions;
- missing publications, software links, or thesis links;
- headshots or bios that should be updated;
- anything on your own project or people bio that you want corrected.

Please pay special attention to pages that directly involve you:

- People pages and bios: https://ubc-fresh.github.io/fresh-lab-web-site/people/
- Project pages: https://ubc-fresh.github.io/fresh-lab-web-site/projects/
- Publications: https://ubc-fresh.github.io/fresh-lab-web-site/publications/

If you find something, please post it as a GitHub issue in the website repo:

https://github.com/UBC-FRESH/fresh-lab-web-site/issues

If you have not used GitHub issues before, the basic workflow is:

1. Open the issues page.
2. Click **New issue**.
3. Give it a short title, such as `Typo on Jamie project page` or `Update my bio`.
4. In the description, include the page URL and explain what should change.
5. Add screenshots if the issue is visual.
6. Submit the issue.

Maintainer notes and website workflow docs are here:

https://github.com/UBC-FRESH/lab-knowledge/wiki/FRESH-Lab-Website-Maintainer-Workflow

You do not need to know how the website build system works to file an issue. Plain-language issue reports are very helpful. If something is easier to describe with a screenshot, just attach the screenshot and write a short note.

Thanks for helping with this. A lab website is public infrastructure: it is how students, collaborators, funders, future trainees, and random internet people first see what FRESH is doing. Getting the details right matters.

Greg

## LinkedIn Draft

I am happy to share that the new website for the FRESH Lab at UBC is live:

https://ubc-fresh.github.io/fresh-lab-web-site/

FRESH is the Forest Resources and Ecosystem Services Hub in the UBC Faculty of Forestry & Environmental Stewardship. Our work focuses on forest-sector systems modelling, operations research, open-source decision-support tools, forest carbon and ecosystem-service modelling, wildfire and disturbance planning, and applied analytics for sustainable forest management.

The new site gives a better public home for our current people, projects, publications, open-source software, thesis work, and collaborations. It is also now backed by a version-controlled GitHub workflow, which should make it much easier for us to keep the site current as the lab grows.

If you are interested in forest modelling, ecosystem services, forest carbon, open-source research software, or decision support for complex forest-management problems, please take a look:

https://ubc-fresh.github.io/fresh-lab-web-site/

Feedback is welcome, and we are always happy to hear from potential students, collaborators, and partners whose interests overlap with FRESH.
